## Base de données

### Sauvegarde

D'après la documentation de [RT](https://docs.bestpractical.com/rt/4.4.4/backups.html#PostgreSQL).

La base de données est sauvegardée tous les jours dans `/root/dumps/`, chaque dump étant horodaté à la minute :

Mot de passe de la base PostgreSQL : **/root/.pgpass**

	localhost:5432:rtdb:rtuser:database_password

Script de sauvegarde : **/root/scripts/rt-dump-database**

	#!/bin/bash
	
	PG_DUMP='/usr/bin/pg_dump'
	DB_USER='rtuser'
	DB_NAME='rtdb'
	DUMP_REP='/root/dumps'
	FILE_NAME="/$DUMP_REP/$DB_NAME-`date +%Y%m%d-%H%M`.sql.gz"
	
	# Dump
	($PG_DUMP -U $DB_USER -h localhost --table=sessions --schema-only $DB_NAME; \
	$PG_DUMP -U $DB_USER -h localhost --exclude-table=sessions $DB_NAME) \
	| gzip > $FILE_NAME
	
	# Keep latest 15 dumps
	find $DUMP_REP -type f -mtime +14 -exec rm {} \;

Entrée dans la *crontab* :

	# grep database /etc/cron.d/request-tracker4
	[...]
	30 0 * * * root /root/scripts/rt-dump-database

On conserve les 15 derniers dumps.

Pour effectuer un backup ponctuel : `sudo /root/scripts/rt-dump-database`

### Restoration

Pour restorer, il faut utiliser le script **rt-setup-database** qui ne fonctionne en postgresql qu'avec un mot de passe (il ne sait pas utiliser l'auth ident unix). On rentre donc un password unique pour le script.

	# sudo -u postgres psql
	postgres=# \password
	Enter new password: 
	Enter it again: 
	# systemctl stop apache2
	# rt-setup-database --action drop 
	In order to create or update your RT database, this script needs to connect to your  Pg instance on localhost (port '') as postgres
	Please specify that user's database password below. If the user has no database
	password, just press return.
	
	Password: <le même>
	...
	# rt-setup-database --action create
	<pareil>
	# gunzip < /root/dumps/rtdb-xxx.sql.gz | sudo -u postgres psql rtdb
	# systemctl restart apache2

### Archivage

Les données de sauvegarde sont archivées sur [[infra/machines/jabba]] via le script **/root/scripts/rt-archive** :

	#!/bin/bash

	RSYNC=`which rsync`
	OPTIONS="-a --delete-after"
	REMOTE="jira@jabba.fdn.fr"

	# Archive database dumps
	$RSYNC $OPTIONS /root/dumps $REMOTE:~

	# Archive attachments
	$RSYNC $OPTIONS /var/lib/request-tracker4/data/attachments $REMOTE:~

	# Archive conf
	$RSYNC $OPTIONS /etc/request-tracker4 $REMOTE:~/etc

L'archivage est lancé tous les jours après la sauvegarde :

	root@jira:~# crontab -l
	# m h  dom mon dow   command
	40 0 * * * /root/scripts/rt-archive

## Apache 2

Si ce n'est pas déjà fait : `a2enmod ssl`

Configuration du site :

	        ServerName tickets.fdn.fr
	        Include /etc/apache2/include/redirect-to-https.conf
	        Include /etc/apache2/include/acme-challenge.conf
	</VirtualHost>
	
	<VirtualHost *:443>
	        ServerAdmin adminsys@fdn.fr
	        ServerName tickets.fdn.fr
	        DocumentRoot /var/www/html
	
	        SSLEngine on
	        SSLCertificateFile      /etc/apache2/ssl/tickets.fdn.fr/tickets.fdn.fr.chained
	        SSLCertificateKeyFile   /etc/apache2/ssl/tickets.fdn.fr/tickets.fdn.fr.key

**51-dbconfig-common.pm** :

	# THE DATABASE:
	# generated by dbconfig-common
	
	# map from dbconfig-common database types to their names as known by RT
	my %typemap = (
	    mysql   => 'mysql',
	    pgsql   => 'Pg',
	    sqlite3 => 'SQLite',
	);
	
	Set($DatabaseType, $typemap{pgsql} || "UNKNOWN");
	
	Set($DatabaseHost, 'localhost');
	Set($DatabasePort, '');
	
	Set($DatabaseUser , 'rtuser');
	Set($DatabasePassword , 'database_password');
	
	# SQLite needs a special case, since $DatabaseName must be a full pathname
	my $dbc_dbname = 'rtdb'; if ( "pgsql" eq "sqlite3" ) { Set ($DatabaseName, '' . '/' . $dbc_dbname); } else { Set ($DatabaseName, $dbc_dbname); }

## Rotation des logs

fichier **/etc/logrotate.d/request-tracker** :

	/var/log/request-tracker4/*.log {
	        daily
	        missingok
	        rotate 14
	        compress
	        delaycompress
	        notifempty
	        create 660 root www-data
	        sharedscripts
	        postrotate
	                if /etc/init.d/apache2 status > /dev/null ; then \
	                    /etc/init.d/apache2 reload > /dev/null; \
	                fi;
	        endscript
	        prerotate
	                if [ -d /etc/logrotate.d/httpd-prerotate ]; then \
	                        run-parts /etc/logrotate.d/httpd-prerotate; \
	                fi; \
	        endscript
	}
